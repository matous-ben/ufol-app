# ADR-004: Maven vs Gradle

## Status
Accepted

## Kontext
Při vývoji Java aplikace je nutné zvolit build tool —
nástroj který spravuje závislosti, řeší kompatibilitu 
verzí knihoven a zajišťuje sestavení aplikace do 
spustitelného JAR souboru. Manuální správa závislostí 
bez build toolu by byla pro projekt této komplexity 
neudržitelná. Byly zvažovány dvě standardní možnosti:

- **Maven** — build tool a správce závislostí založený 
  na XML konfiguraci v souboru `pom.xml`. Dlouhá historie 
  v enterprise Java světě, výchozí volba na start.spring.io, 
  dominuje v Spring Boot ekosystému a vzdělávání.

- **Gradle** — moderní build tool používající Groovy 
  nebo Kotlin DSL v souboru `build.gradle`. Výrazně 
  rychlejší díky inkrementálnímu buildu — přebuilduje 
  pouze změněné části projektu. Dominuje v Android 
  vývoji a moderních open-source projektech.

Oba nástroje jsou plně podporovány Spring Initializr 
(start.spring.io) i IntelliJ IDEA.

## Rozhodnutí
Zvolen Maven.

## Důvody
- Maven je výchozí volba na start.spring.io — odráží 
  jeho dominantní postavení v Spring Boot ekosystému
- Převážná většina Spring Boot dokumentace, tutoriálů 
  a Stack Overflow odpovědí používá Maven a pom.xml — 
  při debugování problémů je snazší najít relevantní 
  řešení
- XML syntaxe pom.xml je pro první projekt přehlednější 
  než Gradle DSL, který vyžaduje znalost Groovy 
  nebo Kotlin syntaxe
- Rychlostní výhoda Gradle díky inkrementálnímu buildu 
  je při projektu této velikosti zanedbatelná
- Maven poskytuje veškeré funkce které projekt vyžaduje — 
  správu závislostí, definici Spring Boot parent POM 
  a sestavení do spustitelného JAR souboru

## Důsledky

**Pozitivní:**
- Snadnější orientace v dokumentaci a komunitních 
  zdrojích — většina příkladů používá Maven
- Standardizovaná struktura projektu kterou rozpozná 
  každý Java vývojář
- Spring Boot parent POM automaticky řeší kompatibilitu 
  verzí závislostí

**Negativní:**
- XML konfigurace je verbose oproti Gradle DSL — 
  pom.xml bývá delší než ekvivalentní build.gradle
- Gradle je rychlejší při buildu a je standardem 
  v moderních projektech mimo enterprise sféru
