# ADR-001: Thymeleaf vs React

## Status
Accepted

## Kontext
Z hlediska frontendu jsem řešil otázku jaký přístup z hlediska náročnosti projektu a mých dosavadních zkušeností zvolit. V úvahu připadala buď CSR (client-side rendering) technologie, tedy například React, který soubory renderuje na straně prohlížeče, a nebo SSR (server-side rendering) technologie jako je šablonovací engine Thymeleaf, kdy se HTML vyrenderuje už na serveru. Backend projektu využívá Spring Boot framework.

## Rozhodnutí
Pro tento projekt byl zvolen šablonovací engine Thymeleaf.

## Důvody
Výběr jsem učinil z toho důvodu, že zatím neovládám React, a proto je pro mě nejen z časových důvodů výhodnější mít vše jako jednu ucelenou jednotku - Spring aplikaci. Dále využívám Spring framework, který umožňuje snadnou implementaci šablonovacího engine jako je Thymeleaf. Již mám vytvořený frontend, tudíž mi stačí vložit jen speciální atributy pro pole s daty. V Reactu bych musel vytvořit REST API na backendu, konzumovat JSON odpovědi na frontendu a vytvářet komponenty. Kromě toho je SSR přístup lepší i z hlediska SEO a indexace, v druhém případě bych musel využít k dosažení stejného výsledku další nástroje jako Next.js, což zatím také neovládám.

## Důsledky
Toto rozhodnutí mi umožňuje mít veškerou aplikaci na straně serveru, a tudíž je snadněji nasaditelná a i udržovatelná. Dále můžu využít již existující část frontendu, což mi zkracuje čas potřebný k vývoji aplikace. Díky Spring ekosystému lze využít i další závislosti pro snažší vývoj.
Ztrácím do budoucna možnost, aby měl projekt SPA (Single-page application) feel. Prozatím se nenaučím moderní frontend framework. Nebude možnost aktualizovat jen určité sekce stránky, ale vždy celou stránky při vložení nových dat. Pokud by v budoucnu měla existovat mobilní aplikace, tak pro ní nebude existovat API.
