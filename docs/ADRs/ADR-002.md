# ADR-002: PostgreSQL vs H2

## Status
Accepted

## Kontext
Aplikace potřebuje relační databázi pro persistentní 
ukládání dat ligy — týmů, hráčů, zápasů a výsledků. 
Byly zvažovány dvě možnosti:

**H2** — embedded databáze napsaná v Javě, běží přímo 
uvnitř JVM procesu Spring Boot aplikace. Nevyžaduje 
žádnou instalaci ani konfiguraci. Defaultně běží 
in-memory — veškerá data zaniknou při restartu aplikace. 
SQL dialekt H2 se částečně liší od produkčních databází, 
což může způsobit zákeřné rozdíly mezi vývojovým 
a produkčním prostředím.

**PostgreSQL** — plnohodnotný relační databázový server 
běžící jako samostatný proces nezávislý na aplikaci. 
Vyžaduje instalaci a konfiguraci. Data jsou persistentně 
uložena na disku a přežijí restart aplikace i serveru.

## Rozhodnutí
Zvolena databáze PostgreSQL 18

## Důvody
- PostgreSQL nabízí produkční prostředí, které odpovídá standardnímu vývoji aplikací v reálném světě
- Aplikace vyžaduje zajistit persistenci dat, a je potřeba aby byla data uložena na serveru i po restartu aplikace
- Samostatný databázový server odpovídá reálné třívrstvé architektuře (FE / BE / DB) kterou projekt implementuje a dokumentuje

## Důsledky
**Pozitivní:**
- Znalost PostgreSQL je přímo uplatnitelná v praxi
- Konzistentní prostředí mezi lokálním vývojem a případným produkčním nasazením
- pgAdmin a IntelliJ Database tool umožňují pohodlnou správu a inspekci dat během vývoje

**Negativní:**
- Vyžaduje instalaci PostgreSQL serveru a počáteční konfiguraci (vytvoření databáze, uživatele, oprávnění)
- Lokální vývoj závisí na běžícím PostgreSQL procesu — při jeho pádu aplikace nemůže startovat
